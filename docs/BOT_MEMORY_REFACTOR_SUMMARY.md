### 2. "从聊天生成"按钮状态独立性

#### 之前的问题
```typescript
// 全局状态，所有机器人共享
generating = false;

// HTML中所有按钮都使用同一个状态
<button [disabled]="generating">
  {{ generating ? '生成中...' : '从聊天生成' }}
</button>
```

**问题**：当为Bot1生成记忆时，所有机器人的"从聊天生成"按钮都会被禁用。

#### 现在的解决方案
```typescript
// 独立状态，使用Set跟踪每个机器人
generatingBotIds: Set<number> = new Set();

// 检查特定机器人是否正在生成
isGenerating(botId: number): boolean {
  return this.generatingBotIds.has(botId);
}

// 生成时添加到Set
generateMemory(botId: number): void {
  this.generatingBotIds.add(botId);
  // ... 生成逻辑
}

// 完成后从Set移除
this.generatingBotIds.delete(botId);
```

**效果**：每个机器人的"从聊天生成"按钮状态完全独立
- Bot1 生成中：`[生成中...]` (禁用)
- Bot2 可用：`[从聊天生成]` (可用)
- Bot3 可用：`[从聊天生成]` (可用)

## 技术亮点

### 1. 状态管理优化
使用 `Set<number>` 数据结构管理多个独立状态：
- O(1) 时间复杂度的添加/删除/检查
- 自动去重，避免重复状态
- 内存效率高

### 2. 用户体验优化
- **自动选择**: 加载完成后自动选中第一个机器人
- **状态保持**: 切换机器人时保持编辑状态
- **独立操作**: 每个机器人的操作互不干扰

### 3. 布局优化
- **Flexbox布局**: 灵活的空间分配
- **固定+自适应**: 左侧固定，右侧自适应
- **优雅降级**: 移动端自动切换为上下布局
